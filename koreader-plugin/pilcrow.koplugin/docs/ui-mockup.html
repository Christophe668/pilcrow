Locked decisions

SubtitleCounts and last-sync on the left, a tappable ↻ Sync button on the right (44 px hit target). Tap = trigger sync without opening the menu. Saves two taps vs. hamburger → Sync now.
FooterFive tappable regions: « ‹ first/prev page · centre Page X / Y (tap to jump to a specific page via InputDialog) · › » next/last page. Disabled chevrons dim. This is what KOReader's Menu renders by default — no code needed beyond using Menu, the chevrons auto-show when there are multiple pages and auto-enable/disable based on current page. The date previously shown is dropped.
Row layout (v2)Card style: thumbnail (~120 × 80 native) on the left, two-line title, one meta line "domain · reading time · age". Items per page: 5 (≈251 px native per row, well above the 88 px Kobo touch-target floor).
Image sourceWallabag's preview_picture field. Downloaded during sync, stored at <data_dir>/pilcrow/images/<id>.jpg. Articles without a preview picture get a generic placeholder glyph.
Title typographySerif (matching reader). Bold for unread, regular weight for archived. Truncates with ellipsis after 2 lines.
Meta lineSans-serif, smaller, ghost colour. Domain is left-most for scannability; reading time follows; "added X ago" rounds to days/weeks.
Active-filter displayChip row below the subtitle. Wraps to multi-line when needed. Each chip-row line costs one article-row.
Status chipTap opens a 4-button mini-dialog (Unread / Starred / Archived / All). Marker: ›.
Tag chipsOne chip per active tag. Tap ✕ to remove. Add more from top-bar → Tags….
Search chipShown when a search term is active (⌕ "term" ✕). Tap ✕ to clear.
Sort chipHidden at default ("Newest first"). Shown only when overridden. Tap ✕ to reset to default.
Tag selectionMulti-select with checkboxes. Semantics: intersection — articles must match all selected tags. Apply button commits.
Sort persistencePersisted across sessions in pilcrow.lua.
Tags entry placementTop-bar dialog (no header shortcut). Reason: chip row already exposes active tags.
SearchComposes with filter + tags + sort. Substring on title and domain.

Implementation impact

  • New custom widget: ChipRow (HorizontalGroup of mini ButtonContainers). ~120 lines.
  • cache.lua: extend list() to take a tag set + sort key. tagCounts() helper for the multi-select dialog.
  • queueview.lua: thread tag set, sort, search through _buildItemTable. Render chip row above the menu items when any filter is active.
  • settingsview.lua: add sort_key persisted setting (default "newest").
  • main.lua: no changes beyond passing extra args.
